### v25–v29 · Sesión del 21 de Abril 2026

#### v26 · `533bf5f` · Bug IPS case-sensitivity
**Problema:** El campo `descuento_ips_actual` contenía valores como `"No aplica"` (N mayúscula) ingresados manualmente. El código comparaba con `"no aplica"` exacto → no reconocía la excepción → IPS aparecía al 95% en lugar de 100%.  
**Solución:**
```javascript
// Antes
var isAplicable = (ips !== 'no aplica' && ...);
// Después
var ipsLow = ips.toLowerCase();
var isAplicable = (ipsLow !== 'no aplica' && ...);
```

#### v29 · `645a657` · Fix gráficas rotas + Cuestionario en sidebar
**Problema 1 — Gráficas vacías:**  
`shortSalario(s)` quedó con `return m[s]||s;` después de borrar el cuerpo de la función, pero `m` no estaba definida. Esto causaba un `ReferenceError` que rompía la ejecución de todos los `requestAnimationFrame` de Chart.js, dejando todas las gráficas vacías.  
**Corrección:** `function shortSalario(s){ return s; }` — retorna el nombre completo sin transformar.

**Problema 2 — "Cuestionario" no aparecía en sidebar:**  
La pestaña se intentó insertar como HTML estático, pero el sidebar se genera **dinámicamente** desde el array `tabs`. El elemento no quedó en ese array.  
**Corrección:** Se añadió al array:
```javascript
{id:'survey', icon:'📝', label:'Cuestionario'}
```
Y se registró en el router de tabs:
```javascript
const tabHandlers = { ..., survey: renderSurvey };
```

**Problema 3 — `window.nav` indefinido:**  
El botón "Ir a Base Analítica" llamaba `window.nav('Respuestas')` que no existía como función global.  
**Corrección:** Se creó `window.nav(tabId)` que mapea nombres a IDs de tab y ejecuta la navegación.
